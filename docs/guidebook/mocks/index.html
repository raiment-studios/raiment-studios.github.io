Items in the outline can have a page of notes associated with them. The note format is a set of Markdown-like blocks. Includes extensions for wikiwords, tagging, and data tables.

Refactor the rendering code
The code currently has all the rendering objects in a largeState object. While this was convenient for getting things started (by ignoring code organization), it's becoming a maintenance problem. The code should be split into different modules and structures for the logical parts of the rendering code.
Logical classes to consider:
  • Model
  • Geometry
  • Material
  • Transform
When in doubt, try to follow the spirit of three.js' API design as it is both easy to use and efficient.

Collections​

Sub-trees can optionally be viewed as a table with tags. All the expected search, sort, and filter features work on the table. The parent item can be given a schema to validate values, format them, provide defaults, offer dropdowns or other UX enhancements.
